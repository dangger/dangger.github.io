Shadowsocks新加密方式：RC4-MD5

10月 1, 2014

好久没更新blog了 军训什么的不是很累。。。。= =
不扯淡 说正题
我自己的VPS环境是Centos 至于Debian之类的。。。。好像wiki写的很详细了
因为我是渣渣 之前用了一键安装脚本。。。。所以就干脆直接重新安装了一遍

yum install m2crypto python-setuptools
easy_install pip
pip install shadowsocks
然后配置了/etc/shadowsocks.json文件 clowwindy大大默认还是用了AES-256-FB 所以我们要把这个改一下

{
“server”:”你的VPS的ip”,
“server_port”:端口号,
“local_address”: “127.0.0.1”,
“local_port”:1080,
“password”:”密码”,
“timeout”:300,
“method”:”aes-256-cfb”,
“fast_open”: false,
“workers”: 1
}

把上方method里改为 RC4-MD5
好像小写是不可以的 因为我写小写不知道为啥不能用。。。一定是我太弱TAT
之后保存 一行ssserver -c /etc/shadowsocks.json后就可以啦~
速度还是挺好的 放个图 3.5刀的VPS 日常用足够了 求土豪不要嘲讽QAQ

如果想要让其后台运行 那么我们需要Supervisor
Centos下 安装很简单 yum install supervisor
然后我们要配置一下 这里有个小问题 wiki给的是/etc/supervisor/supervisord.conf 但是我的文件却位于/etc目录下
打开这个文件后我们看到一堆英文 直接在末尾添加
[program:shadowsocks]
command=ssserver -c /etc/shadowsocks.json
autorestart=true
user=nobody
如果端口 < 1024，把上面的 user=nobody 改成 user=root。
保存后 我们需要手动启动一下Supervisor
Centos下
service supervisord start
然后不出意外的话 你现在就可以享受RC4-MD5带来的快感了 速度比起AES-256-FB来讲快了一点点。。。。VPS好渣QAQ
最后还是感谢clowwindy为我们带来了Shadowsocks这样的利器 向他致敬
